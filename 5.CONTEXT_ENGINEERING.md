# CONTEXT ENGINEERING

## Learning Objectives

- Understand different types of context and how they interact
- Optimize context for token efficiency and accuracy
- Design systems that dynamically inject relevant context
- Apply context filtering and compression techniques

---

## Introduction

Context engineering is one of the most underrated skills in prompt engineering. Most people think prompting is just about the instruction — *"do X"* — but the context surrounding that instruction often determines **80% of the output quality**.

Context engineering is the discipline of deciding what information to include, what to exclude, how to structure it, and when to inject it.

---

# Types of Context

## User Context

Who is asking? What are their preferences, history, and goals?

### User Context Example

```text
"The user is a non-technical marketing manager.
Avoid jargon. Explain technical concepts with analogies.
Their company sells B2B HR software."
```

---

## System Context

Overall rules, persona, and operational guidelines.

### System Context Example

```text
"You are a helpful assistant for Acme Corp's customer portal.

You can only discuss topics related to our products.

For all billing questions, direct users to billing@acme.com.

Never discuss competitors."
```

---

## Dynamic Context

Information injected at runtime based on the current situation — retrieved documents, current date, user's current action.

### Dynamic Context Example

```text
Current time: May 20, 2026 at 3:47 PM EST
User's current page: /settings/billing
User's plan: Pro (expires July 1, 2026)
Last support ticket: 14 days ago, resolved
```

---

# Context Prioritization

When your context window is limited, prioritize:

```text
Priority 1: Task-critical instructions (what to do, what NOT to do)

Priority 2: Dynamic current-session context

Priority 3: User-specific personalization

Priority 4: Background knowledge / reference material

Priority 5: Historical conversation data
```

# Context Compression

## What it is

Reducing context length while preserving essential information.

## Techniques

```text
1. Summarization:
Replace long history with summary

Before:
2,000 tokens of conversation

After:
"User is troubleshooting a login error.
Two solutions tried:
password reset (failed),
cache clear (partial success)." → 30 tokens


2. Extraction:
Pull only relevant fields from documents

Before:
Full 10-page report

After:
Only the 3 sections relevant to current question


3. Semantic Filtering:
Remove context with low relevance score

Use embedding similarity to keep only highly relevant chunks
```

# Context Design

## Scenario

You're building a customer service chatbot for an e-commerce platform.

Design the context architecture for a user asking about a delayed order.

## Solution

```text
SYSTEM CONTEXT (always present, 150 tokens):

You are Maya, a helpful customer service agent for ShopFast.

Be empathetic, solution-focused, and brief.

Escalate to human agents for issues you cannot resolve.

Never make promises about delivery dates unless confirmed in order data.


DYNAMIC CONTEXT (injected at runtime, 200 tokens):

Order #SF-2892:
Laptop bag, ordered May 10,
estimated May 15.

Current status:
In transit.
Last scan May 18 in Chicago hub.

Carrier: FedEx #786234897

Customer tier:
Premium (3yr member)

Previous contacts:
0 this order


USER MESSAGE:

"Where is my order? It was supposed to arrive 5 days ago."


RESPONSE STRATEGY:

Acknowledge delay,
provide tracking details,
offer compensation
(Premium members: 10% next order coupon),
give updated ETA.
```
