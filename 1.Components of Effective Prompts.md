# Components of Effective Prompts

A world-class prompt contains up to six key components. Not every prompt needs all six, but understanding each gives you a complete toolkit.

## The Six Components

```text
+------------------------------------------------------+
|                  EFFECTIVE PROMPT                    |
|                                                      |
| 1. ROLE        – Who is the AI?                      |
| 2. CONTEXT     – Background info                     |
| 3. TASK        – What to do                          |
| 4. EXAMPLES    – Show don't tell                     |
| 5. CONSTRAINTS – What to avoid                       |
| 6. FORMAT      – Output structure                    |
|                                                      |
+------------------------------------------------------+
```

---

## 1. Role (Persona Assignment)

Telling the AI who it is activates relevant knowledge and tone.

### Weak

```text
"Explain machine learning."
```

### Strong

```text
"You are a senior data scientist explaining machine learning to a marketing executive with no technical background."
```

---

## 2. Context

Provide background that the AI needs but doesn't have.

### Weak

```text
"Write a follow-up email."
```

### Strong

```text
"Write a follow-up email.

Context:

I met Sarah Chen at the TechCon 2026 conference.
She's VP of Engineering at Stripe.
We discussed integrating their payment API into our SaaS product.
She asked me to send details about our usage volume."
```

---

## 3. Task

Be specific about what you want. Use action verbs.

### Weak

```text
"Help with my presentation."
```

### Strong

```text
"Create a 5-slide executive summary presentation outline covering:

- Problem statement
- Market size
- Our solution
- Competitive advantage
- Funding ask"
```

---

## 4. Examples (Few-Shot)

Show the model what good output looks like.

### Task

```text
Classify customer feedback as Positive, Negative, or Neutral.
```

### Examples

```text
"Love this product!" → Positive
"It's okay, nothing special." → Neutral
"Completely broke after 2 days." → Negative
```

### Now classify

```text
"Faster than I expected but the UI needs work."
```

---

## 5. Constraints

Tell the model what NOT to do, what to limit, and what rules to follow.

### Constraints

```text
- Maximum 100 words
- No technical jargon
- Do not mention competitor names
- Write in present tense only
- US English spelling
```

---

## 6. Format

Specify exactly how the output should be structured.

### Format

Return a JSON object with these fields:

```json
{
  "summary": "string (max 50 words)",
  "key_points": [
    "string",
    "string",
    "string"
  ],
  "action_required": true,
  "priority": "high | medium | low"
}
```
