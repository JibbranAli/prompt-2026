# Prompt Architecture Design

## Single-Prompt Architecture (Simple tasks)

```text
User Input → Single Prompt → Output
```

---

## Pipeline Architecture (Sequential tasks)

```text
User Input
    ↓
Prompt 1 (Extract)
    ↓
Prompt 2 (Analyze)
    ↓
Prompt 3 (Format)
    ↓
Final Output
```

---

## Parallel Architecture (Independent subtasks)

```text
                    User Input
                         ↓
      ┌──────────────────┼──────────────────┐
      ↓                  ↓                  ↓
Prompt 1            Prompt 2           Prompt 3
(Summary)           (Key Facts)       (Action Items)
      └──────────────────┼──────────────────┘
                         ↓
                    Aggregator
                         ↓
                   Final Output
```

---

## RAG Architecture (Knowledge-grounded responses)

```text
Documents
    ↓
Embedding Model
    ↓
Vector Database
    ↑
User Query
    ↓
Query Embedding
    ↓
Similarity Search
    ↓
Top-K Relevant Chunks
    ↓
Prompt = Query + Relevant Chunks
    ↓
LLM Response
```

# Prompt Chaining

## What it is

Using the output of one prompt as the input to the next.

---

## Example Chain — Content Marketing Pipeline

```text
Chain Step 1 — Research

Prompt:
"Given this product: [product],
identify 5 target audience pain points.
Output as JSON array."


Chain Step 2 — Ideation (uses Step 1 output)

Prompt:
"Given these pain points:
[Step 1 output]

Generate 3 blog post titles
that address each pain point.

Format as JSON."


Chain Step 3 — Outline (uses Step 2 output)

Prompt:
"Create a detailed outline for this blog post:
[selected title]

Include:
- Intro
- 5 main sections
- Conclusion

Format as JSON."


Chain Step 4 — Draft (uses Step 3 output)

Prompt:
"Write the full blog post
based on this outline:

[Step 3 output]

Target:
- 1200 words
- SEO optimized
- Casual professional tone"


Chain Step 5 — Polish (uses Step 4 output)

Prompt:
"Review this article:

[Step 4 output]

Check:
- Grammar
- Consistency
- CTA presence
- Keyword density

Return improved version
with change log."
```

# Structured Output with JSON

## The power of JSON prompting

JSON outputs can be directly consumed by applications, parsed programmatically, stored in databases, and chained into future prompts.

---

## JSON Prompt Template

```text
Analyze the following job posting and extract structured data.

Return ONLY valid JSON.

No explanation before or after.

Use null for any field not present.

Required schema:

{
  "job_title": "string",
  "company": "string",
  "location": "string",

  "remote_policy":
    "remote" | "hybrid" | "onsite" | null,

  "salary_range": {
      "min": number | null,
      "max": number | null,
      "currency": string | null
  },

  "required_skills": string[],
  "preferred_skills": string[],
  "years_experience": number | null,
  "education_required": string | null,
  "benefits": string[]
}

Job Posting:

[POSTING TEXT]
```

# Dynamic Prompt Templates with Variables

## What it is

Reusable prompt templates with placeholders filled at runtime.

---

## Template Syntax

```python
TEMPLATE = """
You are a {role} specializing in {domain}.

Your client is a {company_type}
company in the {industry} industry.

They have asked you to {task}.


Key constraints:

- Audience: {audience}
- Tone: {tone}
- Length: {length}
- Format: {format}

Context:

{context}
"""
```

### Runtime Fill

```python
prompt = TEMPLATE.format(

    role="copywriter",

    domain="B2B SaaS marketing",

    company_type="Series B startup",

    industry="HR technology",

    task="write a product launch announcement",

    audience="HR Directors and CHROs",

    tone="professional and exciting",

    length="300 words",

    format="email with subject line",

    context="New AI performance review feature, 40% time savings"
)
```

# Mini Project 3: AI Research Assistant

## System Design

### SYSTEM PROMPT FOR RESEARCH ASSISTANT

```text
You are ResearchAI,
an expert research analyst assistant.


CAPABILITIES:

- Literature synthesis and summarization
- Key finding extraction
- Gap analysis
- Counterargument identification
- Citation formatting


WORKFLOW:

When given research materials:

1. Summarize each source in 2–3 sentences

2. Identify 3–5 cross-source themes

3. Note any contradictions between sources

4. Identify the strongest evidence
   for the main hypothesis

5. List 3 gaps or questions
   for further research


OUTPUT FORMAT:

Use markdown with clear headings.

Include a "Confidence Level"
(High/Medium/Low)
for each major finding.

Always note source limitations.


IMPORTANT:

Never make claims beyond
what sources support.

Flag speculative statements
with [INFERENCE].

Always cite which source
supports each claim.
```
