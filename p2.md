# Prompt Categories

Open-ended prompts allow the AI creative latitude. Best for brainstorming, exploration, and creative work.

> Example:
```text
"What are some innovative ways a retail bank could use AI in 2026?"
```

Closed prompts constrain the AI to a specific type of answer. Best for classification, extraction, and structured outputs.

> Example:
```text
"Classify this email as 'Sales', 'Support', 'Billing', or 'Other'.
Reply with only the category label."
```

Instruction prompts direct task execution. Best for writing, coding, and analysis.

> Example:
```text
"Write a Python function that takes a list of emails and returns only those with valid formats, using regex validation."
```

Context-based prompts are heavily front-loaded with context for grounded responses.

> Example:
```text
"Given the following product description and three customer reviews,
identify the top 3 recurring complaints and suggest improvements.

[PRODUCT]: [description]
[REVIEWS]: [review 1], [review 2], [review 3]"
```

Conversational prompts are designed for multi-turn dialogue and build context across turns.

> Example:
```text
Turn 1: "I'm planning a product launch for a B2B SaaS tool."

Turn 2: "The target audience is HR directors at mid-size companies."

Turn 3: "Help me create a launch email sequence for this audience."
```

# Problem

A product manager wants to generate user story templates for their team.

## Weak Prompt

```text
Write a user story.
```

## Expected Output of Weak Prompt

```text
As a user, I want to log in so that I can access my account.
```

*(Too generic. Completely useless without context.)*

## Strong Prompt

```text
You are an expert product manager specializing in B2B SaaS products.

Create 3 user stories for a "Team Dashboard" feature in a project management tool used by software development teams.

Each user story must follow this format:

- Title: [Short feature name]
- As a [type of user], I want [goal] so that [reason].
- Acceptance Criteria: (3 bullet points)
- Priority: High / Medium / Low
- Story Points: [1-13 Fibonacci scale]

Target users: Scrum Masters, Developers, and Engineering Managers.

Avoid vague language. Be specific about measurable outcomes.
```

# Hands-On Exercise 3.1 — Prompt Deconstruction

## Instructions

Analyze the following prompt and identify each component.

```text
"You are a financial analyst at a hedge fund.

Given the following earnings call transcript,

identify:
(1) the top 3 growth drivers mentioned by management,
(2) any risk factors discussed,
and (3) guidance for next quarter.

Format your answer as a structured report with clear headers.

Do not speculate beyond what's stated in the transcript.

[TRANSCRIPT]: [paste transcript here]"
```

## Answer

- **Role:** "financial analyst at a hedge fund"
- **Context:** earnings call transcript (provided in placeholder)
- **Task:** identify growth drivers, risk factors, guidance
- **Format:** structured report with headers
- **Constraints:** "Do not speculate beyond what's stated"
- **Examples:** implicit (numbered structure shows desired output shape)

---

## Key Takeaways

- 6 components: Role, Context, Task, Examples, Constraints, Format
- Open-ended = creative latitude
- Closed = constrained specific answers
- Always specify output format for structured use cases
- Constraints prevent common failure modes before they happen
