# MULTIMODAL PROMPT ENGINEERING

## Learning Objectives

- Design effective prompts for text, image, and document inputs
- Master image prompting for AI image generation tools
- Extract structured data from documents using prompts
- Combine modalities for richer AI interactions

---

## Introduction

The era of text-only AI is over. Modern AI systems like GPT-4o, Gemini, and Claude can process text, images, audio, documents, and more — sometimes all at once.

Multimodal prompt engineering requires you to think about how different types of inputs interact and how to give the model clear instructions for each modality.

# Text Prompting — Advanced Techniques

## Structure matters at scale

```text
For short prompts (< 100 tokens):

    Natural language works well.


For medium prompts (100–500 tokens):

    Use clear sections with headers.


For long prompts (500+ tokens):

    Use XML-style tags for clear structure:

    <role>Senior software engineer</role>

    <context>Legacy codebase migration project</context>

    <task>Review this Python 2 code for Python 3 compatibility issues</task>

    <code>{code_here}</code>

    <output_format>JSON list of issues with line numbers</output_format>
```

# Image Prompting for AI Image Generation

When prompting image generation models (DALL-E, Midjourney, Stable Diffusion), structure matters enormously.

## Image Prompt Components

```text
[Subject] + [Action/Pose] + [Environment] + [Lighting] +
[Style] + [Camera/Perspective] + [Mood] + [Technical specs]
```

## Example — Weak Image Prompt

```text
"A person working at a desk"
```

## Example — Strong Image Prompt

```text
"A female software engineer in her late 20s working at a dual-monitor standing desk in a modern, plant-filled home office, golden hour afternoon light streaming through floor-to-ceiling windows, focused expression, casual attire, photorealistic style, shot from slightly above eye level, shallow depth of field, warm and productive atmosphere, 8K resolution"
```

## Style modifiers reference

```text
Photography: "photorealistic, DSLR, 35mm lens, bokeh"

Illustration: "digital illustration, vector art, flat design"

Painting: "oil painting, watercolor, impressionist style"

3D Render: "3D render, Cinema 4D, octane render, HDR"

Concept Art: "concept art, ArtStation quality, detailed"
```

# Document Prompting

## PDF/Document Analysis Template

```text
You are a document analyst. I am providing you with [document type].

Your tasks:

1. Executive Summary:
   Summarize in 3–5 sentences

2. Key Data Points:
   Extract all numerical data as a table

3. Action Items:
   List any required actions or decisions

4. Questions:
   List anything ambiguous or requiring clarification


Output format:
Use the section headers above.

Do not add information not present in the document.


[DOCUMENT CONTENT]: {document_text}
```

---

## Resume Analysis Prompt

```text
You are a senior technical recruiter.

Analyze the following resume for a Senior Backend Engineer role
(Python, distributed systems, 5+ years).

Score the candidate 1–10 on:

- Technical Skills Match
- Experience Level
- Project Complexity
- Communication (based on how they describe their work)
- Culture Fit (startup vs enterprise indicators)

For each score, provide 2-sentence justification.

End with:

HIRE / INTERVIEW / PASS recommendation and reasoning.

[RESUME]: {resume_text}
```

# AI Document Intelligence System

## Project Goal

Create a system that can analyze any uploaded business document and extract structured intelligence.

## System Prompt

```text
You are DocuMind, an AI document intelligence system for business professionals.

When a document is provided, automatically:

1. IDENTIFY:
Determine document type
(contract, report, proposal, invoice, etc.)

2. EXTRACT:
Pull all key information relevant to that document type

3. SUMMARIZE:
Write a 3-sentence executive brief

4. FLAG:
Identify anything that requires human attention

5. STRUCTURE:
Output in clean JSON format


JSON Schema:

{
   "document_type": string,

   "summary": string,

   "key_entities": {
      "people": [],
      "companies": [],
      "dates": [],
      "amounts": []
   },

   "critical_flags": [],

   "action_items": [],

   "confidence_score": number (0-1)
}


If the document is unclear or incomplete, list specific questions that would help complete the analysis.
```
