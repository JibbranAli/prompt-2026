# PROMPT OPTIMIZATION AND EVALUATION

## Learning Objectives

- Test and measure prompt performance systematically
- Improve prompts through iterative optimization
- Evaluate accuracy, relevance, and consistency
- Reduce cost while maintaining quality

---

# Introduction

Writing a prompt is only the beginning. Professional prompt engineering involves testing, measuring, and iteratively improving prompts based on evidence.

This is especially critical in production AI applications where thousands of users interact with your prompts daily.

---

# The Prompt Optimization Cycle

```text
1. DEFINE
   What does "good output" look like?
   Define criteria.
          ↓

2. BASELINE
   Run current prompt 10–20 times.
   Record results.
          ↓

3. HYPOTHESIZE
   What specific change might improve output?
          ↓

4. TEST
   A/B test changed version against baseline.
          ↓

5. MEASURE
   Score outputs against defined criteria.
          ↓

6. ITERATE
   Keep what works; discard what doesn't.
          ↓

7. DOCUMENT
   Record the winning prompt and why it works.
```

---

# Evaluation Dimensions

For any AI output, evaluate:

| Dimension | Question | Measurement |
|------------|-----------|-------------|
| Accuracy | Is it factually correct? | Manual spot-check / fact verification |
| Relevance | Does it answer the actual question? | 1–5 scale scoring |
| Completeness | Are all required elements present? | Checklist scoring |
| Format | Does it follow the specified format? | Automated parsing test |
| Consistency | Is it stable across runs? | Run 10x; measure variance |
| Safety | Does it avoid harmful outputs? | Red team testing |

# Practical Optimization Techniques

## Technique 1: Specificity Injection

```text
Before:
"Write a summary."

After:
"Write a 3-sentence summary.

Sentence 1:
main finding

Sentence 2:
key evidence

Sentence 3:
implication"
```

---

## Technique 2: Negative Examples

```text
"Here is an example of what NOT to do:

Bad output:
'There are many factors to consider...'

(This is too vague.
Be specific and data-driven.)"
```

---

## Technique 3: Anchor Phrases

```text
"Start your response with:
'Based on the provided data...'"

"End with:
'In summary, the recommended action is...'"
```

---

## Technique 4: Chain of Quality

```text
"After generating your response:

1. Check:
Have you answered all parts of the question?

2. Check:
Is every claim supported by the provided data?

3. Check:
Is the format exactly as specified?

If any check fails,
revise before returning the final answer."
```

# Cost Optimization

## Token Cost Reduction Strategies

```text
1. Compress system prompts:
Remove redundant instructions

2. Use shorter role descriptions:
"Expert analyst" vs 50-word role

3. Truncate context intelligently:
Keep recent, relevant items

4. Cache common prompts:
Reuse expensive context layers

5. Use smaller models for simple tasks:
GPT-4o-mini for classification

6. Batch processing:
Send multiple items in one call
```

# Prompt Debugging

## Problem

This prompt produces inconsistent outputs.

Find and fix the issues.

### Buggy Prompt

```text
"Analyze this feedback and tell me what customers think about our app.

Also mention the positive things and the negative things.

Be thorough and comprehensive but also concise."
```

---

## Issues identified

1. "Tell me what customers think" → too vague
2. No output format specified
3. "Thorough and comprehensive but concise" → contradictory instruction
4. No role specified
5. No constraints on scope

---

## Fixed Prompt

```text
You are a customer experience analyst.

Analyze the following customer feedback and produce a structured report.

Output format:

POSITIVE THEMES
(top 3, each with a representative quote)

NEGATIVE THEMES
(top 3, each with a representative quote)

NEUTRAL/SUGGESTIONS
(top 2)

OVERALL SENTIMENT:
[Positive / Mixed / Negative]

NET PROMOTER INSIGHT:
[1 sentence]


Constraints:

- Each theme:
  1 sentence description + 1 supporting quote

- Maximum:
  250 words total

- Only include themes with 2+ mentions


[FEEDBACK]:
{feedback_data}
```
